# Achords — For Agents

> Agent integration with achords.

## Start Here

When you start in a repository with `.achords/`:

1. Read `AGENTS.md` in the repo root
2. Read `.achords/AGENTS.md` for org-wide rules
3. Check `.engram/config.json` for repo context

## Reading Order

```
1. .achords/AGENTS.md        # Org rules (main entry point)
2. .achords/config/          # Conventions, policies
3. .engram/config.json       # Repo context
4. repo/AGENTS.md            # Repo-specific rules
```

## Memory

- **Org memory**: `mem_search(project: "ORG_NAME", ...)`
- **Repo memory**: `mem_search(project: "REPO_NAME", ...)`
- **Session end**: Always call `mem_session_summary`

## Skills

Skills follow [Agent Skills spec](https://agentskills.io/specification):

```bash
cat .skills/skills/testing/SKILL.md
```

## Full Documentation

The complete agent documentation is in `.achords/AGENTS.md` — that file is generated by obase and is the source of truth.
